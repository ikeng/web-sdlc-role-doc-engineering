# Standards

## Standards Catalog

This directory contains standards and guidelines for SDLC artifacts and processes.

## Available Standards

| Standard | Category | Purpose |
|----------|----------|---------|
| [Naming Conventions](./naming-conventions.md) | General | File naming, versioning, metadata standards |
| [Code Review Standards](./code-review-standards.md) | Development | Code review criteria and process |
| [Documentation Standards](./documentation-standards.md) | Documentation | Format, style, versioning rules |
| [Quality Gates](./quality-gates.md) | Quality | Review checkpoints and validation criteria |

## Standards Overview

### Naming Conventions
- File naming conventions
- Version numbering scheme
- Metadata and tagging standards
- Directory structure guidelines

### Code Review Standards
- Review criteria and checklist
- Review process and workflow
- Feedback guidelines
- Approval requirements

### Documentation Standards
- Document structure and format
- Writing style guidelines
- Versioning and change management
- Review and approval process

### Quality Gates
- Phase transition criteria
- Artifact quality standards
- Review checkpoints
- Validation requirements

## Application

All SDLC artifacts must comply with these standards:
1. Review relevant standards before creating artifacts
2. Use templates from `../templates/` directory
3. Follow quality gates before phase transitions
4. Maintain consistency across all deliverables

## Maintenance

Standards are reviewed and updated:
- Quarterly by the team
- When new tools or processes are introduced
- When feedback indicates improvements needed
