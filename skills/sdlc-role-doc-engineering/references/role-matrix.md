# Role Matrix

## Roles and Deliverables

| Role | Primary Phase | Deliverables |
|------|---------------|--------------|
| Product Owner | Product | product-vision.md, product-backlog.md, user-stories.md, acceptance-criteria.md |
| Business Analyst | Analysis | requirements-spec.md, use-case-diagram.md, business-rules.md |
| UX Designer | Design | wireframes.md, ui-spec.md, design-system.md |
| Solution Architect | Architecture | architecture-doc.md, tech-stack.md, api-design.md |
| Tech Lead | Development | code-standards.md, dev-guidelines.md, code-review-checklist.md |
| Full-stack Developer | Development | Implementation code, unit tests, integration tests, code review feedback |
| QA Engineer | Quality | test-plan.md, test-cases.md, quality-metrics.md |
| DevOps Engineer | Deployment | deployment-plan.md, ci-cd-pipeline.md, runbook.md |
| Security Engineer | Security | security-requirements.md, threat-model.md, security-checklist.md |
| Project Manager | Project | project-charter.md, sprint-plan.md, risk-register.md |

## Usage

- Use this matrix to select templates for each role's deliverables.
- When creating or enriching role definitions, ensure each role's output files are represented in the project structure.
