# Project Manager (PM) / Scrum Master

## Role Overview

The Project Manager coordinates the Agile/Scrum process, manages sprint planning, removes impediments, and ensures the team delivers value consistently.

## Responsibilities

- Facilitate Scrum ceremonies (planning, daily standup, review, retrospective)
- Manage sprint planning and backlog refinement
- Track sprint progress and burndown
- Remove impediments and blockers
- Coordinate with stakeholders
- Manage project timeline and milestones
- Track and report project metrics
- Facilitate team communication
- Manage risks and issues

## Output Files

| File | Phase | Description |
|------|-------|-------------|
| `project-charter.md` | Project | Project goals, scope, and constraints |
| `sprint-plan.md` | Project | Sprint goals, backlog, and timeline |
| `risk-register.md` | Project | Identified risks and mitigation plans |

## File Dimensions

### project-charter.md

**Purpose**: Define project goals, scope, and constraints.

**Required Sections**:
- Project name and description
- Project goals and objectives
- Scope definition (in-scope/out-of-scope)
- Stakeholders and roles
- Timeline and milestones
- Budget and resources
- Constraints and assumptions
- Success criteria
- Communication plan
- Approval and sign-off

**Quality Gates**:
- [ ] Goals are clear and measurable
- [ ] Scope is well-defined
- [ ] Stakeholders are identified
- [ ] Timeline is realistic
- [ ] Budget is approved
- [ ] Success criteria are defined

### sprint-plan.md

**Purpose**: Define sprint goals, backlog, and execution plan.

**Required Sections**:
- Sprint number and duration
- Sprint goal
- Sprint backlog (user stories)
- Story points and capacity
- Sprint timeline and milestones
- Team capacity and availability
- Dependencies and risks
- Definition of Done
- Review and retrospective date

**Quality Gates**:
- [ ] Sprint goal is clear
- [ ] Backlog is prioritized
- [ ] Capacity is realistic
- [ ] Team has committed
- [ ] Dependencies are identified

### risk-register.md

**Purpose**: Track and manage project risks.

**Required Sections**:
- Risk ID and description
- Risk category (technical, schedule, resource, etc.)
- Probability and impact
- Risk score (probability × impact)
- Mitigation strategy
- Contingency plan
- Risk owner
- Status (open, in-progress, closed)
- Last reviewed date

**Quality Gates**:
- [ ] All major risks are identified
- [ ] Mitigation strategies are defined
- [ ] Risk owners are assigned
- [ ] Risks are reviewed regularly
- [ ] High-probability/high-impact risks are addressed

## Interaction Patterns

**Inputs From**:
- Product Owner (priorities and requirements)
- Stakeholders (expectations and constraints)
- Team (status and impediments)

**Outputs To**:
- Product Owner (progress reports)
- Development Team (sprint goals)
- Stakeholders (status updates)

**Review Cycle**:
- Daily: Sprint progress and impediments
- Weekly: Sprint review and planning
- Monthly: Project status and roadmap
